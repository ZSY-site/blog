    # 面试题汇总

    # 请说一下你对React的理解

    • React是什么
      • React是一个用于【构建用户界面】的 【JavaScript 库】
    • 能干什么，干的怎么样？用途和应用场景
      • 优点：
          1. 编程思想上：可以通过【组件化】的方式能快速构建响应式的大型Web应用程序，各个组件间都是高内聚低耦合的。使用声明式的方式编写
          1. 广泛应用上： 可以使用React开发Web、Android、IOS, 可以实现跨平台。
          1. 社区团队上：开发团队和社区强大。API比较简洁
      • 缺点：
          1. 没有官方系统解决方案：不像vue，就是vue全家桶，想vuex，vue-router都是官方的解决方案。而在react里，很多的react周边库都是需要社区去维护。
          1. 因此给我们带来选型上的麻烦。
    • 如何干的，实现的原理是什么？其它扩展。如果时间充裕，还可以说一下如下的一些原理：
      • JSX实现声明式
      • 虚拟DOM可以实现跨平台
      • React使用的设计模式
      • 自己React大型架构经验

    # 声明式

    let root = document.getElementById('root');

//声明式: 告诉react，我点击h1就会打印hello，内部react怎么做的，我们不需要管
ReactDOM.render(<h1 onClick={()=>console.log('hello')}>hello</h1>,root);


//命令式
let h1 = document.createElement('h1');
h1.innerHTML = 'hello';
h1.addEventListener('click',()=>console.log('hello'));
root.appendChild(h1);

    # 组件化

    • 组件化：把页面拆分为一个个组件，每个组件包含自己的逻辑和样式，再将这些独立组件组合完成一个复杂的页面。这样能方便视图的拆分和复用，还可以做到高内聚和低耦合
    • 每个组件都是高度内聚的，包含自己的样式，数据，逻辑等
    • 各个组件之间又能互相通信，这就是所谓的高内聚低耦合

    # 高内聚低耦合

    • 低耦合：每个模块之间相互联系的紧密程度不高，也就是各个模块独立性很好
    • 高内聚：就是指一个模块内部的各个元素（语句、程序段）之间的联系程度很高

    # React初体验

    <body>

  <div id="test"></div>

  <!-- 引入react核心库 -->
  <script type="text/javascript" src="../js/react.development.js"></script>
  <!-- 引入react-dom，用于操作DOM -->
  <script type="text/javascript" src="../js/react-dom.development.js"></script>
  <!-- 引入babel，用于解析jsx为js -->
  <script type="text/javascript" src="../js/babel.min.js"></script>
  
  <script type="text/babel"> /* 此处必须改为babel，因为我们写的是jsx，需要借助babel翻译 */
    /* 1.创建虚拟DOM */
    //下面这行代码中的<h1>不是html中的<h1>，但是它与html中的<h1>是一一对应的。
    let vDOM = <h1>Hello React!</h1>
    /* 2.借助react把虚拟DOM渲染到页面，变为真实DOM */
    ReactDOM.render(vDOM,document.getElementById('test'))
  </script>

</body>
